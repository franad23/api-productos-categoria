GIT

1. git clone SSH/URL -> Clonar Repo
2. git status estado de cambios (ROJO -> WD, Verde -> Stating)
3. git add . ->  Agrega los archivos al area de Staging
4. git commit -m 'MENSAJE' commitea cambios
5. git checkout -b 'RAMA' -> CREA RAMA
6. git checkout RAMA -> Cambia entre ramas
7. git push origin NOMBRE RAMA Envia cambios a github
8. git pull origin RAMA -> Trae cambios de github segun RAMA
9. git pull origin --rebase RAMA ->  Trae cambios de github segun RAMA
10. git stash -> A los cambio los guarda de manera temporal
11. git stash pop ->  Suelta los cambios del temporal 
!IMPORANTE 
el stash y stash pop se usan normalmente para traer cambios cuando estoy trabajando en un rama
ANTES DE HACER PUSH SIEMPRE HACER UN REBASE y luego la PR
